```text

192.168.56.11 idm2.fis.epn.ec idm2 ldap2.fis.epn.ec ldap2 kdc2.fis.epn.ec kdc2
```

## Deployment order

### idm1

Run as root:

```text
make pki
make pki-demo-certs
bash ldap/scripts/00-install-openldap.sh
make ldap-config-apply
make ldap-certs LDAP_NODE=ldap1
make ldap-enable-ldaps LDAP_NODE=ldap1
make ldap-enable-ldaps-listener
make ldap-syncprov
bash kerberos/scripts/00-install-kerberos.sh
bash kerberos/scripts/01-init-realm.sh
make kerberos-users
make kerberos-services
make kerberos-keytabs
make kerberos-host-keytabs
```

Run as normal user:

```text
make ldap-hash
```

Replace LDAP password placeholders locally before `make ldap-load`.

### idm2

Run as root:

```text
bash ldap/scripts/00-install-openldap.sh
make ldap-config-apply
make ldap-certs LDAP_NODE=ldap2
make ldap-enable-ldaps LDAP_NODE=ldap2
make ldap-enable-ldaps-listener
make ldap-replication-consumer
bash kerberos/scripts/00-install-kerberos.sh
bash kerberos/scripts/06-configure-secondary-kdc.sh
```

Before the consumer command, replace `credentials=REPLACE_WITH_PASSWORD` locally in the LDAP consumer LDIF. Before KDC propagation, copy the correct host keytab to `/etc/krb5.keytab` on each KDC with root ownership and mode 0600.

## Validation

Run as normal user from idm2:

```text
make ldap-search LDAP_URI=ldap://ldap1.fis.epn.ec
bash tests/ldap/test-ldaps.sh ldap1.fis.epn.ec
make kerberos-kinit KRB_USER=jperez
make kerberos-check-propagation
make ha-test
make test-run
```

Expected LDAP result: user entries are returned. Expected LDAPS result: OpenSSL verifies the CA. Expected Kerberos result: `klist` shows a TGT. Expected HAProxy result: LDAP search returns `jperez` through `ldap.fis.epn.edu.ec`.

## Results

`make test-run` records test status in `results/raw/test-results.csv`. Keep the rows generated by the VM tests as evidence. Do not add manually invented results.
